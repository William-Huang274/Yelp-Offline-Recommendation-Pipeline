### 救援式重排（`Stage11`）

`Stage11` 使用同一个 `Qwen3.5-9B` 奖励模型主干，并在 `11-30`、`31-60`、
`61-100` 三个候选区间上训练了三类专家模型。

训练侧证据：

- `11-30` 专家
  - `11-30 true win = 0.9560`
- `31-60` 专家
  - `31-40 true win = 0.7385`
  - `41-60 true win = 0.7605`
- `61-100` 专家
  - `61-100 true win = 0.8626`

`bucket5` 上的评估参考线：

| 结果线 | Recall@10 | NDCG@10 | 说明 |
| --- | ---: | ---: | --- |
| `v120 two-band @ alpha=0.80` | `0.1973` | `0.0898` | 当前 `11-30 + 31-60` best-known offline 结果 |
| `v124 tri-band @ alpha=0.36` | `0.1857` | `0.0838` | 当前 `11-30 + 31-60 + 61-100` 冻结口径 |

当前 tri-band 策略下，深段专家仍按保守方式使用：

- `11-30`：主要负责 top10 边界救援
- `31-60`：主要负责 top10 / top20 局部救援
- `61-100`：更像名次提升路径，而不是激进冲前排

## 这版最值得看的点

1. 这次升级同时覆盖了召回路由、结构化精排和救援式重排三层。
2. 大模型不会对全量候选做重排，而只作用于 `Stage10` 产出的有边界窗口。
3. Stage11 的增益不只是调分数，`11-30`、`31-60`、`61-100` 三类专家都学到了稳定信号。
4. 当前冻结 tri-band 线里的前排收益主要来自 `11-30` 和 `31-60`，`61-100` 先保持保守。
